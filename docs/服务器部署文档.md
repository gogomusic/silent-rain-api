## 1. 系统初始化与 Swap 配置

### 1.1 更新系统软件包

```bash
sudo apt update
sudo apt upgrade -y
```

### 1.2 配置 Swap 空间（内存不足时必备）

当服务器内存较小（如 2GB 或 4GB）时，建议配置 Swap 作为应急内存缓冲。

```bash
# 1. 创建 2GB 的 Swap 文件（可根据实际内存调整大小）
sudo fallocate -l 2G /swapfile

# 2. 设置正确的权限（仅 root 可读写）
sudo chmod 600 /swapfile

# 3. 将文件格式化为 Swap 分区
sudo mkswap /swapfile

# 4. 启用 Swap
sudo swapon /swapfile

# 5. 使 Swap 在系统重启后自动生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 6. 调整 Swappiness 值（建议 10-30，越低越倾向使用物理内存）
sudo sysctl vm.swappiness=10
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf

# 7. 查看 Swap 状态
swapon --show
free -h
```

## 7. 安装与配置 Nginx

### 7.1 安装 Nginx

```bash
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 7.2 创建通用代理配置

```bash
sudo tee /etc/nginx/proxy-common.conf << 'EOF'
proxy_http_version 1.1;
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
proxy_set_header Connection "";

proxy_connect_timeout 30s;
proxy_read_timeout 60s;
proxy_send_timeout 60s;
proxy_buffering on;
proxy_buffer_size 4k;
proxy_buffers 8 16k;
EOF
```

### 7.3 创建站点配置文件

```bash
sudo tee /etc/nginx/sites-available/silent-rain << 'EOF'
server {
    listen 9091;
    server_name <your-server-ip>;

    root /www/silent-rain-admin;
    index index.html;

    # 安全头
    add_header X-Content-Type-Options nosniff always;
    add_header X-Frame-Options SAMEORIGIN always;
    add_header X-XSS-Protection "1; mode=block" always;

    # 上传大小限制
    client_max_body_size 20m;

    # Gzip 压缩
    gzip on;
    gzip_min_length 1024;
    gzip_comp_level 5;
    gzip_types text/plain text/css application/json application/javascript
               text/xml application/xml application/xml+rss text/javascript
               image/svg+xml;
    gzip_vary on;

    # API 代理 -> NestJS 后端
    location /api/ {
        include proxy-common.conf;
        proxy_pass http://127.0.0.1:9161;
    }

    # Swagger JSON (如需导出 API 文档)
    location /api-json {
        include proxy-common.conf;
        proxy_pass http://127.0.0.1:9161/api-json;
    }

    # 文件上传目录 -> 后端静态资源
    location /uploads/ {
        proxy_pass http://127.0.0.1:9161/uploads/;
        include proxy-common.conf;
        expires 30d;
        access_log off;
    }

    # SPA 路由兜底
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源长缓存
    location ~* \.(css|js|woff2?|ttf|otf|eot)$ {
        expires 30d;
        access_log off;
        add_header Cache-Control "public, immutable";
    }

    location ~* \.(jpg|jpeg|png|gif|webp|svg|ico)$ {
        expires 30d;
        access_log off;
    }

    # 日志
    access_log /var/log/nginx/silent-rain-access.log;
    error_log /var/log/nginx/silent-rain-error.log;
}
EOF
```

### 7.4 启用站点

```bash
sudo ln -s /etc/nginx/sites-available/silent-rain /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### 7.5 防火墙与安全组

```bash
sudo ufw allow 9091/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

> **阿里云安全组**：在控制台放行 9091、80、443 端口。

## 8. 安装与配置 Redis

### 8.1 安装 Redis

```bash
sudo apt install redis-server -y
sudo systemctl start redis-server
sudo systemctl enable redis-server
```

### 8.2 安全配置

编辑配置文件 `/etc/redis/redis.conf`：

```bash
sudo vim /etc/redis/redis.conf
```

 **设置密码** （取消注释并修改）：

```
requirepass YourStrong!RedisPass123
```

 **限制内存使用** （建议为总内存的 50%~70%）：

```
maxmemory 2gb
maxmemory-policy allkeys-lru
```

 **绑定地址** （仅内网或本地）：

```
bind 127.0.0.1        # 仅本机访问
# 或 bind 192.168.1.100  # 内网访问
```

重启 Redis：

```bash
sudo systemctl restart redis-server
```

### 8.3 测试连接

```bash
redis-cli -a 你的密码 ping
# 应返回 PONG
```

## 9. 安装与配置 MySQL

### 9.1 安装 MySQL

```bash
sudo apt install mysql-server -y
sudo systemctl start mysql
sudo systemctl enable mysql
```

### 9.2 安全初始化

```bash
sudo mysql_secure_installation
```

> ⚠️ Ubuntu 24.04 默认使用 `auth_socket` 插件，root 只能通过 `sudo mysql` 登录。如需密码登录，先执行：
> ```sql
> ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
> FLUSH PRIVILEGES;
> ```

### 9.3 创建数据库和用户

```mysql
-- 登录（如果未设密码用 sudo mysql）
sudo mysql

-- 创建数据库
CREATE DATABASE silent_rain_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建本地用户
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON silent_rain_db.* TO 'app_user'@'localhost';

-- 如需远程访问，创建远程用户
CREATE USER 'app_user'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON silent_rain_db.* TO 'app_user'@'%';

FLUSH PRIVILEGES;
```

### 9.4 开启远程访问（可选）

修改配置文件 `/etc/mysql/mysql.conf.d/mysqld.cnf`：

```ini
bind-address = 0.0.0.0
```

重启 MySQL：

```bash
sudo systemctl restart mysql
```

阿里云安全组放行 `3306` 端口，并限制访问来源 IP。

## 10. 备份策略

### 10.1 一键备份（推荐）

项目自带了优化过的备份脚本 `backup.sh`，自动处理 Nginx 配置、环境变量、PM2、crontab 等：

```bash
cd /www/silent-rain-api
chmod +x backup.sh
./backup.sh
```

脚本会自动：
- 备份 `/etc/nginx/` 所有配置
- 备份各项目 `.env.production`（不存在则跳过）
- 备份 PM2 dump、crontab、npm 全局包列表
- 打包为 `~/backup/backup_configs_YYYYMMDD_HHMMSS.tar.gz`
- 自动清理 30 天前的旧备份

### 10.2 数据库备份 (MySQL)

```bash
# 手动备份
mysqldump -u app_user -p \
  --no-tablespaces \
  --single-transaction \
  --routines --triggers \
  silent_rain_db > ~/backup/silent_rain_db_$(date +%Y%m%d).sql
```

> ⚠️ `--no-tablespaces` 必须加，否则无 PROCESS 权限会报错。

**定时备份（每日 2:00）**：

```bash
# 创建备份脚本（密码用 mysql_config_editor 安全存储，避免明文）
mysql_config_editor set --login-path=backup --user=app_user --password

sudo tee /usr/local/bin/backup_mysql.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=$HOME/backup/mysql
DATE=$(date +%Y%m%d)
mkdir -p $BACKUP_DIR
mysqldump --login-path=backup \
  --no-tablespaces \
  --single-transaction \
  --routines --triggers \
  silent_rain_db > $BACKUP_DIR/silent_rain_db_$DATE.sql
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
EOF

sudo chmod +x /usr/local/bin/backup_mysql.sh

# 添加 crontab
(crontab -l 2>/dev/null; echo "0 2 * * * /usr/local/bin/backup_mysql.sh") | crontab -
```

### 10.3 恢复示例

```bash
# 恢复 MySQL
mysql -u app_user -p silent_rain_db < ~/backup/silent_rain_db_20260623.sql

# 恢复 Nginx
sudo cp ~/backup/configs_20260623_103000/nginx/sites-available/silent-rain \
        /etc/nginx/sites-available/
sudo nginx -t && sudo systemctl reload nginx

# 恢复完整配置包
tar -xzf ~/backup/backup_configs_20260623_103000.tar.gz -C /tmp/
```

## 12. 故障排查思路

| 问题           | 排查方向                                                              |
| -------------- | --------------------------------------------------------------------- |
| 网站无法访问   | `sudo systemctl status nginx`；阿里云安全组端口；`pm2 list`           |
| 接口报错 502   | 检查后端 PM2 进程：`pm2 logs silent-rain-api --lines 50`              |
| 接口报错 CORS  | 检查后端 `.env.production` 中是否配置了正确的 `CORS_ORIGIN`           |
| 数据库连接失败 | `sudo systemctl status mysql`；`bind-address` 配置；用户权限          |
| Redis 连接失败 | `sudo systemctl status redis`；`redis-cli -a 密码 ping`               |
| 上传文件失败   | 检查 `uploads/` 目录权限：`ls -la /www/silent-rain-api/uploads/`      |
| 内存不足       | `free -h`；Swap 是否生效；Redis `maxmemory`；PM2 `max_memory_restart` |
| 备份脚本报错   | `dos2unix backup.sh` 转换换行符后再试                                 |
| Nginx 启动失败 | `sudo nginx -t` 检查语法；`sudo journalctl -u nginx --no-pager -n 50` |
